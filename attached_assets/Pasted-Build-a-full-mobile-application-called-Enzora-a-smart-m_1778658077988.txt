Build a full mobile application called Enzora — 
a smart medical app for a Smart Wound Patch startup. 
The app connects to a hardware device (ESP32 mini + 
TCS3200 color sensor) through Firebase Realtime 
Database. The ESP32 reads the bandage color via WiFi 
and writes the result to Firebase. The app listens 
to Firebase in real time and alerts the patient.

---

## 🔌 HOW THE SYSTEM WORKS

ESP32 connects to WiFi and writes to Firebase 
Realtime Database at this path:

/sensor
  ├── red: [number]
  ├── green: [number]
  ├── blue: [number]
  └── status: "yellow" | "green" | "blue"

Status meanings:
"yellow" → Wound is safe/normal
"green"  → Early signs of infection
"blue"   → Infection confirmed

The app listens to /sensor/status in real time 
using Firebase onValue listener.
When status changes, the app updates the UI 
and sends alerts immediately.

No Bluetooth. No IP address input. No BLE library.
The only connection between hardware and app 
is Firebase Realtime Database.

## 🔬 COLOR STATUS LOGIC

Listen to Firebase Realtime Database:
const statusRef = ref(db, 'sensor/status')
onValue(statusRef, (snapshot) => {
  const status = snapshot.val()
  // "yellow" | "green" | "blue"
  updateAppStatus(status)
})

Also read RGB values for history:
const sensorRef = ref(db, 'sensor')
onValue(sensorRef, (snapshot) => {
  const data = snapshot.val()
  // { red, green, blue, status }
  saveReading(data)
})

Save every reading to Firestore:
/users/{uid}/wounds/{woundId}/readings/{id}
{ status, red, green, blue, timestamp }

Delete all old threshold/deviation/calibration logic.

## 🗄️ FIREBASE STRUCTURE

Realtime Database (sensor data from ESP32):
/sensor
  ├── red: number
  ├── green: number
  ├── blue: number
  └── status: "yellow" | "green" | "blue"

Firestore (user data):
/users/{uid}
  - name, email, createdAt
  - medicalProfile: {
      age, gender, conditions,
      doctorName, doctorPhone,
      emergencyContact, emergencyPhone
    }

/users/{uid}/wounds/{woundId}
  - name, location, dateAdded, description,
    notes, status, healedAt

/users/{uid}/wounds/{woundId}/readings/{id}
  - status, red, green, blue, timestamp
